# Project Map

This file explains the important folders so new builders know where to look first.

## Root Files

- `BBCLauncher.sln` - Visual Studio solution for the launcher projects.
- `README.md` - short setup and build overview.
- `THIRD_PARTY_NOTICES.md` - license and third-party dependency notes.
- `.gitignore` - excludes generated files, packages, local payloads, logs, and secrets.

## scripts

The `scripts` folder is the main entry point for building and staging.

- `Build-MinecraftJavaTestFixes.ps1` - recommended package build script for the current working native Xbox/UWP path.
- `Build-MesaUwpRuntime.ps1` - builds and stages the Mesa UWP runtime into `native\mesa-uwp`.
- `Build-NativeBridge.ps1` - builds native bridge pieces such as GLFW/OpenGL/OpenAL shims.
- `Get-MinecraftFiles.ps1` - stages Minecraft version files and assets into `local-test-files`.
- `New-FabricProfile.ps1` - creates a Fabric profile from a staged Minecraft version.
- `New-ForgeProfile.ps1` - creates a Forge profile, mainly useful for older versions such as 1.12.2.
- `New-NeoForgeProfile.ps1` - creates a NeoForge profile.
- `New-MultiVersionPayload.ps1` - combines staged version folders into one payload.
- `Build-XboxPackage.ps1` - builds the older C# launcher shell path.

## native

The `native` folder contains the code that makes Java Minecraft run inside the UWP/Xbox environment.

- `corewindow-probe` - current working native CoreWindow host and package project.
- `xbox-glfw` - GLFW behavior adapted for UWP/Xbox.
- `xbox-opengl` - OpenGL-related bridge code.
- `xbox-openal` - OpenAL/audio bridge code.
- `graphics-bridge` - graphics bridge helpers.
- `win-shims` - Windows compatibility shims used by the native runtime.
- `mesa-uwp` - staged Mesa runtime output. This folder is generated locally and should not be committed.

## src

The `src` folder contains the older C# UWP launcher shell.

- `src\BBCLauncher` - C# app project, package manifest, launcher configuration, and UI shell.

This path is useful for launcher UI work, configuration ideas, and future integration. For the working game host package, use `native\corewindow-probe`.

## local-test-files

This folder is created by scripts and should stay out of git.

It can contain:

- staged Minecraft files
- downloaded assets
- Java runtime files
- generated payloads
- bundled local mods
- generated signing certificates

Do not commit this folder.

## artifacts

This folder is generated by build scripts and should stay out of git.

It contains built `.msix` packages and copied build output.

Do not commit this folder.
